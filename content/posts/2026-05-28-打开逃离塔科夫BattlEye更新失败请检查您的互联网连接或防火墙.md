## 问题描述

电脑打开《逃离塔科夫》时，游戏启动日志优先出现以下错误：

```text id="s6c8xm"
13:28:34: 启动 BattlEye 服务...
13:28:35: 更新中...
13:28:45: 更新失败 (1, 28)。 请检查您的互联网连接和 / 或防火墙。
13:28:48: 启动游戏...
13:29:16: 无法启动游戏。
```

随后系统弹出报错：

```text id="2xjv9h"
Windows 无法访问指定设备、路径或文件。你可能没有适当的权限访问该项目。
```

游戏无法正常启动，BattlEye 反作弊服务更新失败。

## 问题原因

排查过程中发现，Windows 防火墙并未直接导致该问题。

进一步检查 Windows 安全中心后，发现电脑存在病毒或恶意程序记录。部分恶意程序可能修改 Hosts 文件，导致 BattlEye 反作弊服务相关地址无法正常连接，从而出现更新失败 `(1, 28)` 报错。

此外，系统文件损坏、权限异常或 Windows 快速启动功能，也可能导致反作弊服务无法正常加载。

## 解决办法

### 1. 临时关闭 Windows 防火墙测试

打开：

```text id="j7k1r5"
控制面板 → Windows Defender 防火墙 → 启用或关闭 Windows Defender 防火墙
```

临时关闭专用网络和公用网络防火墙后，重新测试游戏。

如果问题解决，可进入：

```text id="4r5q0f"
允许应用或功能通过 Windows Defender 防火墙
```

将 Steam、BattlEye 和游戏程序加入白名单。

如果关闭无效，建议恢复默认防火墙设置。

---

### 2. 使用 Windows 安全中心进行全盘扫描

打开：

```text id="s3q4mr"
Windows 安全中心 → 病毒和威胁防护
```

查看“保护历史记录”，确认近期是否存在病毒、木马或风险程序。

随后执行：

```text id="a0tb98"
扫描选项 → 完整扫描
```

也可以使用第三方安全软件进行全盘查杀。

---

### 3. 使用 SFC 修复系统文件

以管理员身份打开命令提示符，输入：

```cmd id="zv0n2x"
sfc /scannow
```

等待系统自动扫描并修复损坏文件。

---

### 4. 修复 Hosts 文件

打开目录：

```text id="0m8lq3"
C:\Windows\System32\drivers\etc
```

找到 `hosts` 文件并提前备份。

由于系统目录权限较高，可先将文件复制到桌面等低权限目录后再编辑。

使用记事本打开后，检查是否存在异常域名解析记录。

删除异常内容后保存，再复制回原目录覆盖原文件。

---

### 5. 关闭 Windows 快速启动

打开电源设置，关闭：

```text id="j2w5fa"
启用快速启动
```

保存设置后重启电脑。

---

### 6. 重启电脑并重新测试游戏

完成杀毒、Hosts 修复和系统修复后，重启电脑。

重新打开 Steam 和《逃离塔科夫》进行测试，BattlEye 报错消失，游戏恢复正常启动。
