## 问题描述

启动《三角洲行动》或运行约两三分钟后出现报错弹窗：

安全功能提示

⚠️ 为有效防范与检测外挂行为，本次启动游戏需调用 CPU 虚拟化技术（VT-X/D 或AMD-V）。
检测到您的设备当前未启用该技术，或存在其他程序（如旧版模拟器、第三方沙盒软件）独占该资源，请配合开启，共同维护公平的游戏环境。

运行机制说明
本安全防护技术仅在游戏进程运行期间生效。当您启动游戏并进入环境后，防护系统将同步开启，在游戏内存层建立硬件级隔离屏障，为您提供持续的外挂防护与安全监测；该技术不会对系统运行效率及稳定性产生不利影响。
当您正常退出游戏后，防护系统将随之终止运行，不执行任何后台常驻进程，亦不占用系统资源或影响您的日常设备使用；您也可以在游戏关闭后撤销相关系统设置。

操作说明
请重启设备进入 BIOS，开启 VT-x/d（Intel）或 AMD-V 选项，并在 Windows 设置中完成相应配置，建议同步关闭可能占用该通道的模拟器或沙盒软件。
点击「确定」可跳转至完整操作指引页面。我们理解本次操作存在一定门槛，感谢您对共同守护公平竞技环境的配合与支持。
如选择「取消」游戏将会关闭，并无法启动游戏。

## 问题原因

游戏安全防护机制依赖 CPU 虚拟化技术（VT-x / AMD-V）实现内存级隔离检测，当环境不满足时将阻止启动，常见原因包括：

* BIOS 未开启虚拟化（VT-x / AMD-V / SVM）
* IOMMU 等相关功能未正确启用
* Hyper-V 或系统 Hypervisor 占用虚拟化资源
* 模拟器、虚拟机或沙盒软件冲突占用
* Windows 安全策略限制（内存完整性、设备加密）
* 系统环境或文件异常

## 解决办法

### 一、修复系统环境

1. 以管理员身份打开命令提示符
2. 执行：

   ```
   sfc /scannow
   ```
3. 等待扫描完成

### 二、校准系统时间

1. 打开“调整日期和时间”
2. 开启自动时间与自动时区
3. 点击“立即同步”（time.windows.com）

### 三、关闭设备加密

1. 设置 → 隐私和安全性
2. 关闭“设备加密”

### 四、禁用虚拟化占用

1. 打开：

   ```
   appwiz.cpl
   ```

2. 进入“启用或关闭 Windows 功能”

3. 取消勾选 **Hyper-V** 及相关组件

4. 以管理员身份执行：

   ```
   bcdedit /set hypervisorlaunchtype off
   ```

### 五、排查冲突软件

1. 卸载或关闭：

   * 模拟器
   * 虚拟机软件
   * 第三方沙盒或安全软件

2. 精简启动服务：

   ```
   msconfig
   ```

   * 勾选“隐藏所有 Microsoft 服务”
   * 禁用异常服务

### 六、关闭系统安全限制

1. 打开“Windows 安全中心”
2. 执行病毒扫描并处理问题
3. 关闭：

   * 内存完整性
   * 智能应用控制

### 七、开启 BIOS 虚拟化

1. 重启进入 BIOS

2. 启用：

   * Intel：VT-x / VT-d
   * AMD：SVM Mode

3. AMD 平台：

   * Advanced → AMD CBS → IOMMU → Enable

4. 保存并退出

### 八、修复游戏文件

1. 右键游戏 → 修复
2. 优先快速修复
3. 异常时执行深度修复

### 九、重启验证

1. 重启电脑
2. 启动游戏测试
3. 确认不再出现虚拟化报错
