## 问题描述

运行《Delta Force》（三角洲行动）时，游戏启动后通常可以正常进入，但在**跳伞阶段**或游玩几分钟后会弹出安全提示并强制关闭游戏。

原始报错内容：

> **安全功能提示：**
> ⚠️ 为有效防范与检测外挂行为，本次启动游戏需调用 CPU 虚拟化技术（VT-X/D 或 AMD-V）。检测到您的设备当前未启用该技术，或存在其他程序（如旧版模拟器、第三方沙盒软件）独占该资源，请配合开启，共同维护公平的游戏环境。
>
> **运行机制说明**
> 本安全防护技术仅在游戏进程运行期间生效。
>
> **操作说明**
> 请重启设备进入 BIOS，开启 VT-x/d（Intel）或 AMD-V 选项，并在 Windows 设置中完成相应配置，建议同步关闭可能占用该通道的模拟器或沙盒软件。
>
> 如选择「取消」游戏将会关闭，并无法启动游戏。

---

## 问题原因

该报错通常并非 CPU 不支持虚拟化，而是以下原因导致：

1. BIOS 未开启虚拟化功能

   * Intel：`VT-x / VT-d / VMX`
   * AMD：`AMD-V / SVM`

2. `IOMMU` 未正确开启（部分 AMD 主板常见）

3. `Hyper-V` 占用了虚拟化资源

4. 模拟器、沙盒软件、虚拟机等程序独占虚拟化功能

5. Windows 安全策略冲突

   * 内存完整性
   * 内核隔离
   * 智能应用控制

6. 游戏文件异常或缺失

7. 系统时间同步异常导致安全校验异常

## 解决办法

### 1、扫描并修复系统文件

右键开始菜单 → 打开 **终端（管理员）**

执行：

```bash id="gmw6g7"
sfc /scannow
```

等待扫描完成。

---

### 2、同步系统时间

进入：

**设置 → 时间和语言 → 日期和时间**

确保以下选项开启：

* 自动设置时间
* 自动设置时区
* 自动同步时间

检查时间服务器为：

```bash id="gm16g8"
time.windows.com
```

点击 **立即同步**。

---

### 3、关闭设备加密（微软账户用户）

进入：

**设置 → 隐私和安全性 → 设备加密**

如果使用微软账户登录 Windows，建议先关闭设备加密，避免修改 BIOS 后无法正常进入系统。

本地账户可跳过。

---

### 4、卸载占用虚拟化的软件

按 `Win + R`

输入：

```bash id="9skjo8"
appwiz.cpl
```

检查并卸载：

* 安卓模拟器
* 虚拟机软件
* 沙盒工具
* 第三方安全软件

---

### 5、关闭 Hyper-V

进入：

**控制面板 → 启用或关闭 Windows 功能**

取消勾选：

* Hyper-V

完成后重启电脑。

---

### 6、关闭内存完整性

打开 **Windows 安全中心**

进入：

**设备安全性 → 内核隔离**

关闭：

* 内存完整性

---

### 7、关闭智能应用控制

进入：

**应用和浏览器控制**

关闭：

* 智能应用控制

---

### 8、进入 BIOS 开启虚拟化

重启电脑，开机连续按 `Delete` 进入 BIOS。

#### AMD 平台

进入：

`Advanced → CPU Configuration`

开启：

`SVM Mode → Enabled`

---

#### Intel 平台

开启：

* VT-x
* VT-d
* VMX

---

### 9、AMD 平台开启 IOMMU

部分 AMD 主板还需额外设置：

`Advanced → AMD CBS → IOMMU`

如果当前为：

`Auto`

修改为：

`Enabled`

保存退出 BIOS。

---

### 10、修复游戏文件

打开 WeGame

找到《Delta Force》

右键选择：

* 修复游戏

如果检测到文件异常或缺失：

* 使用深度修复

修复完成后重新启动游戏。
