## 问题描述

老版本 Windows 10 在系统重置后，进入 OOBE（开箱体验）阶段时，界面强制要求连接网络并登录微软账户。
在无网络环境或不希望使用微软账户的情况下，无法继续下一步安装流程。

## 问题原因

部分老版本 Windows 10（如 1903 之前版本）在 OOBE 阶段默认优先引导用户联网并登录微软账户，本地账户入口被隐藏。
当检测到网络可用时，系统会强制进入联网验证流程，导致无法直接创建本地账户。

## 解决办法

### 方案一：使用 oobe\bypassnro.cmd（推荐）

适用于多数版本。

步骤：

1. 在强制联网界面断开所有网络（拔网线或关闭 WiFi）。

2. 按 `Shift + F10`（笔记本可尝试 `Fn + Shift + F10`）打开 CMD。

3. 输入命令：

   ```
   oobe\bypassnro.cmd
   ```

4. 回车后系统自动重启。

5. 重启后进入 OOBE 界面，出现“我没有 Internet 连接”选项。

6. 选择该选项，即可创建本地账户。

---

### 方案二：本地账户直达命令

步骤：

1. 断开网络。

2. 打开 CMD。

3. 输入命令：

   ```
   start ms-cxh:localonly
   ```

4. 回车后弹出本地账户创建窗口。

5. 完成设置后进入桌面。

---

### 方案三：结束联网进程

适用于老版本 Windows 10。

步骤：

1. 断开所有网络。

2. 打开 CMD。

3. 输入命令：

   ```
   taskkill /F /IM oobenetworkconnectionflow.exe
   ```

4. 回车后联网界面关闭，跳转至本地账户创建界面。

5. 填写用户名并完成设置。

---

### 方案四：禁用网卡方式

步骤：

1. 断开网络并打开 CMD。

2. 输入命令：

   ```
   devmgmt.msc
   ```

3. 在“网络适配器”中禁用所有网卡。

4. 关闭窗口后选择“我没有 Internet 连接”。

5. 创建本地账户。

6. 进入桌面后重新启用网卡。
