## 问题描述

打开 Steam 客户端无任何界面显示，程序长时间转圈无响应。
查看日志未发现明确报错信息。
重新安装过程中提示：**“已经在电脑上运行”**，但系统中实际已卸载。

## 问题原因

1. Steam 卸载后存在残留文件或注册表信息。
2. 网络配置异常（IPv6 未启用、DNS 配置异常或缓存未刷新）。
3. 系统时间不同步导致验证异常。
4. hosts 文件存在未注释的异常映射。
5. 原安装目录残留导致程序误判正在运行。

## 解决办法

### 一、彻底卸载并清理残留

1. 打开“应用和功能”，找到 Steam。
2. 执行卸载并完成卸载流程。
3. 删除残留的 Steam 文件夹。
4. 清理相关注册表信息。

### 二、修复网络配置

1. 右键 Windows 开始菜单，选择“终端（管理员）”。

2. 输入：

   ```
   ncpa.cpl
   ```

3. 打开当前正在使用的网络适配器。

4. 进入“属性”：

   * 勾选 **IPv6**。
   * 将 IPv4 DNS 恢复默认或设置为 `8.8.8.8`。

5. 保存设置。

6. 在管理员终端输入：

   ```
   ipconfig /flushdns
   ```

   刷新 DNS 缓存。

### 三、检查系统时间

1. 打开系统时间设置。
2. 确认时间与当前实际时间一致。
3. 如不一致，执行时间同步。

### 四、检查 hosts 文件

1. 打开目录：

   ```
   C:\Windows\System32\drivers\etc
   ```

2. 使用记事本打开 `hosts` 文件。

3. 检查是否存在未加 `#` 注释的异常地址映射。

4. 如存在异常条目，将其注释或删除后保存。

### 五、重新安装 Steam

1. 重新运行 Steam 安装程序。
2. 若提示“已经在电脑上运行”，重启电脑。
3. 开机后再次运行安装程序。
4. 更改安装路径（例如改为 D 盘）。
5. 修改安装文件夹名称，例如改为 `Steam2`。
6. 等待更新完成。
7. 安装完成后关闭客户端，再手动重新打开一次确认。

完成上述步骤后，Steam 客户端可正常启动。
