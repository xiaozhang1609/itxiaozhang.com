## 问题描述

启动《三角洲行动》时出现以下安全提示：

> **安全功能提示**
>
> ⚠️ 为有效防范与检测外挂行为，本次启动游戏需调用 CPU 虚拟化技术（VT-X/D 或 AMD-V）。检测到您的设备当前未启用该技术，或存在其他程序（如旧版模拟器、第三方沙盒软件）独占该资源，请配合开启，共同维护公平的游戏环境。
>
> **运行机制说明**
>
> 本安全防护技术仅在游戏进程运行期间生效。当启动游戏并进入环境后，防护系统将同步开启，在游戏内存层建立硬件级隔离屏障，为持续的外挂防护与安全监测提供支持。
>
> **操作说明**
>
> 请重启设备进入 BIOS，开启 VT-x/d（Intel）或 AMD-V 选项，并在 Windows 设置中完成相应配置。建议同步关闭可能占用该通道的模拟器或沙盒软件。
>
> 如选择“取消”，游戏将关闭，并无法启动。

部分用户即使进入《三角洲行动》，也可能在跳伞阶段或游玩过程中再次弹出该报错。

---

## 问题原因

该问题通常由以下原因导致：

* BIOS 未开启 CPU 虚拟化功能
* Hyper-V 占用 CPU 虚拟化资源
* Windows 安全功能占用虚拟化通道
* 模拟器、虚拟机、沙盒软件冲突
* 第三方安全软件干扰
* 系统时间异常
* 游戏文件损坏
* 系统文件异常导致虚拟化调用失败

## 解决办法

### 1、修复系统文件

右键 Windows 开始菜单，选择 **以管理员身份运行命令提示符**。

输入以下命令：

```bash id="quy0w4"
sfc /scannow
```

等待扫描完成。

---

### 2、关闭 Hyper-V 启动项

系统扫描完成后，继续在管理员命令提示符中输入：

```bash id="vdf9l7"
bcdedit /set hypervisorlaunchtype off
```

执行完成后如果提示：

`操作成功完成`

说明 Hyper-V 默认启动项已关闭。

---

### 3、同步系统时间

进入：

**设置 → 时间和语言 → 日期和时间**

开启：

* 自动设置时间
* 自动设置时区
* 自动同步时间

确认时间服务器为：

`time.windows.com`

然后点击 **立即同步**。

---

### 4、关闭设备加密（如有）

进入：

**设置 → 隐私和安全**

如果使用微软账户登录，并存在 **设备加密** 功能，建议关闭。

---

### 5、卸载占用虚拟化的软件

按 `Win + R` 输入：

```bash id="6nqlza"
appwiz.cpl
```

检查并卸载以下软件：

* 模拟器
* 虚拟机软件
* 沙盒软件
* 第三方安全软件

---

### 6、关闭 Hyper-V 功能

进入：

**启用或关闭 Windows 功能**

找到：

**Hyper-V**

如果已勾选：

* 取消勾选
* 保存设置

未启用则无需处理。

---

### 7、检查异常服务

按 `Win + R` 输入：

```bash id="5eqo43"
msconfig
```

进入 **服务**：

* 勾选 **隐藏所有 Microsoft 服务**
* 检查异常第三方服务
* 暂时禁用可疑服务

---

### 8、检查区域设置（海外系统可选）

按 `Win + R` 输入：

```bash id="2r6dyg"
intl.cpl
```

进入：

**管理 → 更改系统区域设置**

如游玩国服游戏，可根据实际情况调整设置。

---

### 9、关闭无关启动项

打开任务管理器：

进入 **启动应用**

关闭无关的开机启动程序。

---

### 10、关闭内存完整性

打开 **Windows 安全中心**

进入：

**设备安全性 → 内核隔离**

关闭：

**内存完整性**

---

### 11、在 BIOS 中开启 CPU 虚拟化

重启电脑进入 BIOS（通常开机连续按 `Delete`）。

#### AMD 处理器

确保以下选项开启：

* SVM Mode
* IOMMU

#### Intel 处理器

确保以下选项开启：

* VT-x / VMX
* VT-d

保存设置并重启电脑。

---

### 12、修复《三角洲行动》游戏文件

打开 WeGame：

找到《三角洲行动》

按顺序执行：

* 快速修复
* 若提示文件损坏，再执行深度修复

修复完成后重新启动游戏。
