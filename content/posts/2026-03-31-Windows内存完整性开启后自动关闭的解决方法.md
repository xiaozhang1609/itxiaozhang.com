## 问题描述

在 Windows「Windows 安全中心 → 设备安全性 → 内核隔离详细信息」中开启内存完整性后，重启电脑发现功能自动变回关闭状态，反复尝试均无法保持开启。

## 问题原因

内存完整性（内核隔离）依赖 CPU 硬件级安全功能 NX/DEP（数据执行保护）。当该功能未在 BIOS/UEFI 中启用时，系统检测到关键依赖缺失，会自动关闭内存完整性。

同时，CPU 虚拟化（Intel VT-x / AMD-V）和安全启动（Secure Boot）未开启，也可能导致该功能无法正常启用或运行不稳定。

## 解决办法

### 步骤 1：进入 BIOS/UEFI 设置界面

开机时反复按对应快捷键进入 BIOS：

* 联想 / 雷神：F2 或 Fn+F2
* 戴尔：F2 或 F12
* 华硕：Del 或 F2
* 惠普：F10
* 微星：Del 或 F2

或通过系统进入：

设置 → 系统 → 恢复 → 高级启动 → 立即重启 → 疑难解答 → 高级选项 → UEFI 固件设置

---

### 步骤 2：开启 NX/DEP 功能

在 BIOS 中找到以下选项并设置为 Enabled：

* Intel：No-Execute Memory Protection / Data Execution Prevention (DEP) / NX
* AMD：No Execute Page Protection / NX Bit / Enhanced Virus Protection

常见位置：

* Advanced（高级）
* Security（安全）
* CPU Configuration（CPU 配置）

---

### 步骤 3：开启虚拟化与安全启动

确保以下功能已开启：

* 虚拟化

  * Intel：Intel VT-x / Virtualization Technology
  * AMD：AMD-V / SVM Mode

* 安全启动

  * Secure Boot → Enabled

---

### 步骤 4：保存并重启

按 F10 保存 BIOS 设置并退出，系统自动重启。

---

### 步骤 5：重新开启内存完整性

进入：

Windows 安全中心 → 设备安全性 → 内核隔离详细信息

开启「内存完整性」，重启后再次检查状态。

---

### 验证与排查

若仍自动关闭，可继续检查：

* 是否存在未签名或老旧驱动（通过“查看内核隔离兼容性信息”）
* 是否安装冲突软件（旧版虚拟机、第三方杀毒、部分反作弊程序）
* Windows 版本是否支持相关功能
